BERT-PoC1-Demo

Tensorflow Serving된 BERT Model들에 Inference 요청을 보내거나,
BERT Model의 Fine-tuning 요청을 보낼 수 있는 Demo App 입니다.

BERT-PoC1 Structure

Inference

각 모델에 대한 inference 요청은 다음과 같은 순서로 진행됩니다.

Demo - Preprocessor container - BERT Model TF-Serving container

Demo에서 입력받은 sentence1, sentence2를 preprocessor로 보내 tokenizing 한 후 output을 그대로 model serving container로 보내서 결과를 받아옵니다. 결과 값을 tf.argmax()로 처리한 후 값이 1이라면 equivalent, 0이라면 not equivalent를 띄워줍니다.

general은 pretrain model, glue/mrpc는 glue/mrpc dataset을 이용해 fine tuning된 model에 요청을 보냅니다. Demo와 각 container간의 통신은 gRPC를 이용해서 이루어집니다.

Fine tuning

fine tuning 을 요청하면 runContainer API를 실행시켜 lg-registry에 저장된 FineTuning docker image를 실행합니다. 실행 후 fine tuning된 모델은 자동으로 gitlab과 s3 저장소에 저장됩니다.

  • USER_ID, USER_PASSWORD : gitlab 접근을 위한 user id 와 password
  • MODEL NAME, MODEL VERSION : pretrained model의 이름과 버전
  • NEW MODEL VERSION: fine tuning된 새로운 모델의 버전
  • EPOCHS : 훈련 반복 횟수 (default=3)
  • BATCH SIZE : 훈련 시 사용되는 batch size (default=32)
  • DATASET : tensorflow dataset에서 불러올 dataset의 이름 (default = glue/mrpc).

glue/mrpc : Microsoft Research Paraphrase Corpus (Dolan & Brockett, 2005)는 온라인 뉴스 소스에서 자동으로 추출 된 문장 쌍의 모음으로, 쌍의 문장이 의미 상 동일한 지 여부에 대한 사람 주석이 포함되어 있습니다. (출처 : https://www.tensorflow.org/datasets/catalog/glue#gluemrpc)

DEV > Build Requirements

  • python3
  • flask
  • tensorflow

or

  • docker

DEV > Environment Variables

$ export BASE_URL = <BASE_URL> # base model
$ export FINETUNE_URL = <FINETUNE_URL> # finetuned model
$ export TOKEN_URL = <TOKEN_URL> # preprocessor
$ export API_URL = <API_URL> # api server
$ export AWS_ACCESS_KEY_ID = <AWS_ACCESS_KEY_ID>
$ export AWS_SECRET_ACCESS_KEY = <AWS_SECRET_ACCESS_KEY> # s3 접근을 위한 key

docker 환경에서 진행한다면 .env 파일을 만들어주세요.

# .env
BASE_URL = <BASE_URL> # base model
FINETUNE_URL = <FINETUNE_URL> # finetuned model
TOKEN_URL = <TOKEN_URL> # preprocessor
API_URL = <API_URL> # api server
AWS_ACCESS_KEY_ID = <AWS_ACCESS_KEY_ID>
AWS_SECRET_ACCESS_KEY = <AWS_SECRET_ACCESS_KEY> # s3 접근을 위한 key

DEV > Run

$ python3 app.py

localhost:80 으로 접속할 수 있습니다.

docker로 실행하려면

$ docker build -t ${IMAGE_NAME} .
$ docker run -it -p 80:80 --env-file .env ${IMAGE_NAME}

마찬가지로 localhost:80 으로 접속할 수 있습니다.
